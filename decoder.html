Как работает модуль Decoder

Декодер начинает работу с получения фрейма от демодулятора.
Рассмотрим пример:

1100100101111110010001000000001001111110110000000101011111100111010010

Первые 24 символа переводятся в десятичное число. Это называется Синхрослово. Затем, на основании этого значения, из конфигурации ConfigDrill определяется тип фрейма и его структура.
Пример конфигурации:

"Frames": [
    {"Type": 0, "Sync": 3250782, "crc": 1, "rd": [42, 43, 44, 45, 46, 47, 3]},
    {"Type": 1, "Sync": 7085514, "crc": 0, "rd": [6, 179, 2, 44, 40, 41, 181]},
    {"Type": 3, "Sync": 13205060, "crc": 0, "rd": [224, 3, 2, 5]}
]

Для определения длины каналов используется информация о каналах:

"Dpu": [
    {"ID": 224, "rL": 8},
    {"ID": 2, "rL": 13},
    {"ID": 3, "rL": 9},
    {"ID": 6, "rL": 10}
]

Пример фрейма:

  • Sync = 13205060
  • Канал 1 = 2
  • Канал 2 = 253
  • Канал 3 = 4117
  • Канал 4 = 31977
